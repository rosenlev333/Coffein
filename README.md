# Coffein 👁️

Маленькое приложение для строки меню macOS (Apple Silicon — M1/M2/M3/M4 …),
которое не даёт Mac засыпать. Под капотом — системная команд­а `caffeinate -dimsu`.

- **Значок-глаз** в строке меню: открытый глаз = режим включён, закрытый = выключен.
- **Левый клик** по значку — мгновенно включить/выключить режим.
- **Правый клик** — меню: Настройки, «Запускать при входе», Выход.
- Держит Mac бодрым **даже после блокировки экрана** `Cmd+Ctrl+Q`.
- Две независимые настройки:
  1. **Запускать при входе** — приложение само появляется в строке меню при старте компьютера.
  2. **Сразу включать режим при запуске** — после старта режим «не спать» включается сам.

## Как это работает

При включении Coffein запускает дочерний процесс
`/usr/bin/caffeinate -dimsu -w <PID>` и создаёт системные power-assertions:

| флаг | что делает |
|------|------------|
| `-d` | не давать гаснуть дисплею |
| `-i` | не давать системе уходить в простой-сон |
| `-m` | не давать засыпать диску |
| `-s` | не давать системе спать (только при питании от сети) |
| `-u` | объявить пользователя активным |

Флаг `-w <PID>` страхует от «зависшего» caffeinate: он сам завершится, как только
процесс Coffein исчезнет (даже при аварийном завершении). При выключении Coffein
дополнительно посылает caffeinate сигнал на завершение — все assertions снимаются мгновенно.

## Ограничения (честно)

- **Закрытие крышки ноутбука всё равно усыпит Mac** — это аппаратное ограничение macOS,
  `caffeinate` его не обходит. Когда крышка открыта и экран заблокирован — держит полностью.
- Флаг `-s` (запрет сна системы) действует **только при питании от сети**. На батарее
  держится запрет простоя (`-i`), но система может уснуть в энергосберегающих сценариях.
- Принудительный сон (меню  → «Режим сна», кнопка питания) по-прежнему работает.

## Требования

- macOS 13+ (Ventura и новее), Apple Silicon.
- Установленные Command Line Tools (`xcode-select --install`) — Xcode не нужен.
- Python 3 с Pillow только для генерации иконки (`python3 -m pip install Pillow`).

## Сборка и установка

```bash
# собрать .app и установить в ~/Applications
bash scripts/build_app.sh --install

# запустить
open ~/Applications/Coffein.app
```

Приложение подписывается ad-hoc-подписью (`codesign -s -`) — этого достаточно для
локального запуска и регистрации в «Объекты входа». После каждой пересборки автозапуск,
возможно, придётся включить заново (Системные настройки → Основные → Объекты входа).

## Проверка ядра без графики

```bash
swift build -c release
bash scripts/self_test.sh
```

Скрипт включает caffeinate, печатает `pmset -g assertions`, выключает и проверяет,
что assertions сняты. Ожидается `RESULT: PASS`.

```
Sources/Coffein/

  AppDelegate.swift          строка меню, клики, меню, жизненный цикл

  Preferences.swift          флаг «включать при запуске»

  SettingsWindowController.swift / SettingsView.swift   окно настроек (SwiftUI)
scripts/
  make_icon.py               генерация цветной иконки «глаз»
  build_app.sh               сборка .app + подпись + установка
  self_test.sh               headless-проверка
```
